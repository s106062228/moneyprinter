## H7: Smart Clipping Module

### Component: `src/smart_clipper.py` (NEW)

**Purpose**: Detect scene boundaries in video files using PySceneDetect, transcribe audio via faster-whisper (already in requirements.txt), score transcript segments for engagement via Ollama LLM, and output ranked clip metadata.

**Design Decisions**:
- Standalone module in `src/`, not a class in `src/classes/` — it's a utility, not a platform integration.
- Returns clip metadata (start/end times, scores, transcript) — does NOT do the actual ffmpeg split. Splitting is a future step.
- Uses existing `llm_provider.generate_text()` for LLM scoring — provider-agnostic.
- Uses `faster-whisper` (already in requirements.txt) for local transcription — no new STT dependency.
- PySceneDetect is the only new pip dependency.

**Architecture**:

```
Input (video file path)
    │
    ├─ 1. Scene Detection (PySceneDetect ContentDetector)
    │      → List of (start_time, end_time) scene tuples
    │
    ├─ 2. Audio Transcription (faster-whisper)
    │      → List of (start, end, text) segments
    │
    ├─ 3. Segment Merging
    │      → Merge scenes + transcript into coherent segments
    │      → Each segment has: time range, transcript text, scene count
    │
    ├─ 4. LLM Engagement Scoring (llm_provider.generate_text)
    │      → Prompt: "Rate this transcript segment 1-10 for virality..."
    │      → Parse score from LLM response
    │      → Each segment gets an engagement score
    │
    └─ 5. Ranked Output
           → List of ClipCandidate objects sorted by score
           → Each has: start, end, duration, score, transcript, reason
```

**Classes**:

```python
@dataclass
class ClipCandidate:
    start_time: float          # seconds
    end_time: float            # seconds
    duration: float            # seconds
    score: float               # 0.0-10.0 engagement score
    transcript: str            # text content
    reason: str                # LLM explanation of score
    scene_count: int           # number of scene cuts in segment

class SmartClipper:
    def __init__(self, min_clip_duration: float = 15.0,
                 max_clip_duration: float = 60.0,
                 top_n: int = 5,
                 whisper_model: str = "base"):
        ...

    def detect_scenes(self, video_path: str) -> list[tuple[float, float]]:
        """Use PySceneDetect ContentDetector to find scene boundaries."""

    def transcribe(self, video_path: str) -> list[tuple[float, float, str]]:
        """Use faster-whisper to get timestamped transcript segments."""

    def merge_segments(self, scenes, transcript) -> list[dict]:
        """Merge scene boundaries with transcript into coherent segments."""

    def score_segments(self, segments: list[dict]) -> list[ClipCandidate]:
        """Use LLM to score each segment for engagement potential."""

    def find_highlights(self, video_path: str) -> list[ClipCandidate]:
        """Full pipeline: detect → transcribe → merge → score → rank."""
```

**Config Integration**:
- `smart_clipper_min_duration` (default 15.0s) in config.json
- `smart_clipper_max_duration` (default 60.0s) in config.json
- `smart_clipper_top_n` (default 5) in config.json
- Whisper model reuses existing `whisper_model` config key.

**Dependencies**:
- NEW: `scenedetect[opencv]` in requirements.txt
- EXISTING: `faster-whisper` (already in requirements.txt)
- EXISTING: `llm_provider.generate_text()` (already in src/)

## H8: Selenium Test Environment Fix

### Component: `tests/conftest.py` (MODIFY)

**Purpose**: Add a session-scoped autouse fixture that pre-mocks selenium and related heavy imports so that all test files can import YouTube/Twitter classes without import-time crashes.

**Current Problem**:
- `test_twitter_youtube_cache.py` has 35 failing tests because importing `Twitter`/`YouTube` classes triggers `selenium`, `moviepy`, `assemblyai` imports at module level.
- The file already tries to handle this with `sys.modules` patching + `__new__`, but the approach is fragile and breaks.

**Fix Strategy**:
1. Add a `conftest.py` fixture that pre-patches `sys.modules` with mocks for all heavy dependencies BEFORE any test file imports them.
2. Use `pytest.importorskip` pattern for tests that actually need real Selenium.
3. Make the mock fixture session-scoped and autouse so it applies globally.

**Changes Needed**:

In `tests/conftest.py`, add:
```python
@pytest.fixture(autouse=True, scope="session")
def mock_heavy_imports():
    """Pre-mock heavy dependencies that crash on import without system packages."""
    mocked_modules = [
        "selenium", "selenium.webdriver", "selenium.webdriver.common",
        "selenium.webdriver.common.by", "selenium.webdriver.common.keys",
        "selenium.webdriver.support", "selenium.webdriver.support.ui",
        "selenium.webdriver.support.expected_conditions",
        "selenium_firefox",
        "webdriver_manager", "webdriver_manager.firefox",
        "moviepy", "moviepy.editor", "moviepy.video", "moviepy.video.fx",
        "moviepy.video.fx.all", "moviepy.video.fx.all.crop",
        "moviepy.video.tools", "moviepy.video.tools.subtitles",
        "moviepy.config_defaults",
        "assemblyai",
        "instagrapi",
    ]
    mocks = {}
    for mod in mocked_modules:
        if mod not in sys.modules:
            mocks[mod] = MagicMock()
            sys.modules[mod] = mocks[mod]
    yield
    for mod, mock in mocks.items():
        if sys.modules.get(mod) is mock:
            del sys.modules[mod]
```

In `tests/test_twitter_youtube_cache.py`:
- Remove the manual `sys.modules` patching boilerplate (now handled by conftest).
- Simplify imports to directly use `from Twitter import Twitter` etc.
