## H12: Smart Clipper Video Splitting

### Component: `src/smart_clipper.py` (MODIFY)

**Purpose**: Add `split_clips()` method that converts ClipCandidate metadata to PySceneDetect scene_list format and calls `split_video_ffmpeg()` for actual clip extraction.

**Design Decisions**:
- Method added to existing SmartClipper class (not a new module)
- Uses PySceneDetect's built-in `split_video_ffmpeg()` — no direct subprocess calls
- Returns list of output file paths for caller to use
- Configurable output directory and filename template
- Validates ffmpeg availability before attempting split

**New Method**:

```python
def split_clips(
    self,
    video_path: str,
    candidates: list[ClipCandidate],
    output_dir: str = ".",
    filename_template: str = "$VIDEO_NAME-clip-$SCENE_NUMBER",
) -> list[str]:
    """
    Split video into clips based on ClipCandidate metadata.

    Converts ClipCandidates to PySceneDetect scene_list format,
    then calls split_video_ffmpeg() for fast lossless extraction.

    Args:
        video_path: Path to source video file.
        candidates: List of ClipCandidates from find_highlights().
        output_dir: Directory for output clips.
        filename_template: Filename pattern ($VIDEO_NAME, $SCENE_NUMBER, $START_TIME, $END_TIME).

    Returns:
        List of output clip file paths.

    Raises:
        FileNotFoundError: If video_path doesn't exist.
        RuntimeError: If ffmpeg is not available.
    """
```

**Implementation Flow**:

```
Input: video_path + list[ClipCandidate]
    │
    ├─ 1. Validate inputs (file exists, ffmpeg available)
    │      → is_ffmpeg_available() check
    │
    ├─ 2. Create output directory (os.makedirs)
    │
    ├─ 3. Convert ClipCandidates → scene_list
    │      → Each candidate → (FrameTimecode(start), FrameTimecode(end))
    │      → Sort by start_time for sequential processing
    │
    ├─ 4. Call split_video_ffmpeg()
    │      → split_video_ffmpeg(video_path, scene_list, output_dir, filename_template)
    │
    └─ 5. Collect output paths
           → Glob output_dir for generated files matching template
           → Return sorted list of paths
```

**Dependencies**:
- `scenedetect.video_splitter.split_video_ffmpeg` (already installed)
- `scenedetect.video_splitter.is_ffmpeg_available` (availability check)
- `scenedetect.frame_timecode.FrameTimecode` (time conversion)

### Test Plan

10+ unit tests covering:
1. Basic split with mocked ffmpeg
2. Empty candidates list → returns empty
3. FileNotFoundError for missing video
4. RuntimeError when ffmpeg unavailable
5. Output directory creation
6. Custom filename template
7. Candidates sorted by start time before splitting
8. Return value matches expected file paths
9. Integration with find_highlights() output format
10. Edge case: single candidate

All tests mock `split_video_ffmpeg` and `is_ffmpeg_available` — no actual ffmpeg calls.
